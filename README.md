# 点子开关知识库助手

这是一个基于 Dify 搭建的企业知识库聊天助手 Demo。它用于演示如何把企业介绍、产品服务、价格套餐、售后规则、合同付款、项目流程和常见问题整理成知识库，并通过聊天方式让用户快速查询。

## 产品定位

点子开关知识库助手面向企业内部员工、客服、销售、运营和管理人员，帮助团队把分散在文档、表格和流程说明里的信息变成可问答的知识服务。

它的核心目标是：

- 让用户不用翻文件也能快速找到答案。
- 让客服和销售保持统一、准确的回复口径。
- 让企业知识库的内容更容易被测试、维护和迭代。
- 演示 Dify 在企业知识库场景中的应用方式。

## 主要功能

### 企业信息查询

用户可以查询点子开关的企业介绍、品牌定位、服务范围、品牌口号和对外介绍话术。

示例问题：

- 你们公司是做什么的？
- 为什么叫点子开关？
- 你们适合服务什么类型的企业？

### 产品与套餐咨询

助手可以回答关于产品服务、套餐区别、适用客户、交付内容、交付周期和起步价格的问题。

示例问题：

- 啪嗒知识库助手轻装版多少钱？
- 认真版和轻装版有什么区别？
- 醒醒数据看板适合什么客户？
- 流程审批卡住适合哪个产品？

### 售后与服务流程查询

助手可以根据知识库回答售后支持范围、响应时效、投诉处理流程和问题升级规则。

示例问题：

- 紧急故障多久响应？
- 投诉多久给处理意见？
- 售后支持范围包括哪些内容？

### 合同、付款与发票咨询

助手可以回答合同签署方式、付款节点、发票类型、开票信息和退款规则。

示例问题：

- 是否支持开票？
- 项目付款节点是什么？
- 项目启动前退款能退多少？

### 项目交付流程查询

助手可以介绍从需求沟通、资料整理、系统配置、测试优化到培训上线的完整项目流程。

示例问题：

- 项目上线前要经历哪些阶段？
- 测试优化阶段会交付什么？
- 客户需要提前准备哪些资料？

## 回答原则

该助手被设置为严格根据知识库内容回答问题。

- 知识库中有明确答案时，优先按知识库回答。
- 知识库中没有明确答案时，不编造、不猜测。
- 涉及价格、合同、退款、发票、法律、财务、隐私和安全事项时，只引用知识库已有内容。
- 对复杂或不确定问题，建议联系人工确认。
